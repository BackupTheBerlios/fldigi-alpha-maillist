Joseph A. Counsil wrote:
>  Greetings,
>
> I have been using the released version 3.20.28 today, and found a bug 
> with the Esc key function.  I'm running WinXP Pro/SP3/latest updates.
>
> When I click on the TUNE button, with previous versions I was able to 
> hit the Esc key on the keyboard to abort the TUNE tone.  Also, with 
> previous version when I do an auto CQ (a macro with the TIMER 
> function), and the timer is counting between transmissions, I can 
> abort the timer with a tap of the Esc key on the keyboard.  I found 
> that with released version 3.20.28 this does not work - the Esc 
> keypress is ignored during TUNE and TIMER functions.  However, with 
> all versions if I press the Esc key WHILE text is being transmitted, 
> it will abort the current transmission.  I also tried 3.20.281 in the 
> Alpha directory, and it has the same problem with the Esc key being 
> ignored during TUNE and TIMER functions.
>
> So, I reverted to 3.21.0AK, and the Esc key will abort any function in 
> any mode at any time (including the TUNE function) as expected.  This 
> Esc key abort always worked in previous Alpha and Released versions 
> that I have tried.
>
> 73,
> -Joe-
>
Changes were made in the way that the ESC key functions while 
transmitting.  The reason was to prevent a loss of data when autosending 
data from either flmsg or flarq.  I will look to see what is required to 
restore the functionality to the Tune and Macro timer functions.  In the 
meantime I would recommend using 3.20.28 for the benefit of other 
changes that are not in 3.21.0AK.

73, Dave
