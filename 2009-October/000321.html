Rick Kunath wrote:
> On Friday 23 October 2009 9:40:28 pm w1hkj wrote:
>   
>> Version 3.13AW
>>     wfdisp-mod
>>         * Restore upper limit on waterfall display
>>     
>
> The waterfall upper limit doesn't seem to be working right for me. 
>
> It always expands to fill the max width it can on the window. I get over 5k now 
> on x1.
>   
That is the correct behavior on x1, x2 or x4; if and only if the display 
width is larger than the upper limit that you have set.
The FFT display will not be shown above the upper limit setting.
If the display width is less than the upper limit then the scroll higher 
will not display above the upper limit.
> x2 and x4 seem to set the limit to the top value if and only if, the width of 
> the displayed waterfall exceeds the screen width that Fldigi can fit on the 
> waterfall.
>
> If it doesn't, it just extends the upper limit of the scale off until it fills 
> it.
>   
Correct.  see explanation above.
> Shouldn't the waterfall expand in size to max the window width with the max 
> value, i.e. if I set it to 3k then go from 0 to 3k on x1 and appropriate 
> expansion on the x2 and x4 settings?
>   
The design of the waterfall / spectrum display is such that the 
relationship between FFT bins and pixels is an integer.  Scaling the 
waterfall to fit what ever pixel width the display was set to would 
significantly increase the cpu load.  That window is refreshed 8x / second.
