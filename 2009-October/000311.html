Walter,

This is the same situation which I reported earlier, and is caused by 
the waterfall width in Hz being much wider than the window width.  This 
is why I asked Dave to restore the parameter so the user can set the 
waterfall width in Hz.  In older versions, you could set that parameter 
so that the waterfall would not slide back and forth in the window when 
clicking on the QSY button.

This is not really a "problem" with frequency error or anything like 
that, but is just a user-interface nuisance, which requires extra effort 
to re-center the waterfall.

Dave said that he is working on restoring the waterfall width parameter.
