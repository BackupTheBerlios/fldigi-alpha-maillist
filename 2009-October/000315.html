Known bug - rig-qso minimized log does not clear the RSTout when the 
clear button is pressed.

The restore upper limit on frequency display is not a reversion to the 
older code.  Changing upper limit does not require a restart of the 
program to become effective.  You can see the changes immediately on the 
waterfall or the spectrum display.  The scale and scale markers now 
extend the full width of the waterfall.  Most obvious if you use in full 
screen mode on a large display.  Moved the configuration for these items 
under the waterfall / FFT processing tab:



I would welcome suggestions as to where best to place these 
configuration items.
