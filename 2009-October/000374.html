We have been talking with the hamlib developers and found a solution to
this issue.  The next public release of fldigi will no longer load other
hamlib dlls found on the system.

For the curious: we are simply using versioned symbols.  Fldigi will
only load hamlib libraries that contain the same versions as the ones it
was built with.  Advanced users can still build their own hamlib with the
right ABI_VERSION, or of course build their own fldigi binary.
