Flrig is a separate rig control program that can be also run as an 
adjunct to fldigi on Linux and Windows platforms (no OS X yet).  It 
currently supports the following transceivers

1.  FT100D          11. IC756PRO-II         21. TT516
2.  FT450           12. IC756PRO-II         22. TT538
3.  FT817           13. IC7000              23. TT550
4.  FT857D          14. IC7200              24. TT563
5.  FT2000          15. IC7700
6.  IC706MKIIG      16. IC910H
7.  IC728           17. K2
8.  IC735           18. K3
9.  IC746           19. TS140
10. IC746PRO        20. TS2000

The program configures its primary dialog in accordance with the 
capability of the transceiver.  The user only sees and can operate the 
controls that are relevant to the transceiver being controlled.  
Controls not available are either hidden or grayed out.



Older Icom Model
	

Modern Icom HF/VHF all mode


Low cost Yaesu HF all mode
	

TenTec - Pegasus - Computer Only Transceiver


If you use the program with fldigi, then fldigi needs to start first and 
finish last as it acts as a socket server to the flrig program.  When 
used together flrig sends vital data to fldigi respecting the 
transceiver characteristic, TRANSCEIVER MODES / TRANSCEIVER BANDWIDTHS.  
For some transceivers the bandwidth selection is mode dependent.  flrig 
manages that relationship.  You tell fldigi to work with flrig by 
selecting the rig control called "xmlrpc" from the rig control 
configuration tabs.

Most, but not all of the supported transceivers have been tested.  
Remember that you will be testing alpha code if you choose to try this 
program.

The s/n and power out meters will be active if and only if your 
transceiver supports sending that data.
