Dave,

In general, it looks good.

About CPU usage, I see that AR runs around 10% and AS runs about twice 
that at 20%.

I don't know if the doubling of usage is due to the wider waterfall (I 
run 3000 Hz in AS), but I prefer being able to run 3000 due to 
user-interface layout reasons, and the action of the QSY button.  With 
the wider 4000 Hz waterfall, when I QSY, the whole waterfall slides to 
the left (this is because I have my window width less than the 4000 Hz 
width).  I prefer to be able to set it at 3000 Hz so it won't slide when 
I QSY.  It seems reasonable to keep the waterfall width as a 
user-settable parameter.

I'm running Windows XP Pro, SP3.

Thanks & 73,
-Joe-
