Still no joy getting RigCAT to work with my Kenwood TS-570
using fldigi-3.13AJ under OS X.  To be sure I was setting it up
correct I did the following:

Read the docs at

<http://www.w1hkj.com/FldigiHelp/TransceiverControl.html>

and setup the RigCAT configuration panel to match the
example then downloaded a fresh rigCAT xml file at

<http://www.w1hkj.com/xmlarchives.html>

and once it was downloaded and installed in my .fldig/rigs folder I noticed
the downloaded file extension was not displayed. I did the OS X Get Info and
unchecked the box to hide the extension. This is a very important step for
any Mac user because fldigi will not function without file extensions.

After getting everything set, I clicked on the Save and the red Initialize
button.  I could tell fldigi was talking because the CTRL was lit on the
TS-570S display but the frequency in the fldigi display was wrong and
did not change with the VFO.   Fldigi generated an Event log saying
over and over again:

   W: rigCAT_getmode: Retries failed

   W: rigCAT_getfreq: Retries failed

I tried several configuration changes and could not clear the
error.  Something is not correct so I went back to using hamlib
that works. 

I would like to know if there is another Kenwood TS-570 fldigi user that
can try the test on another platform.  I am lost as to what may be causing
the communications problem.

73 Dick AA5VU

Ps: Other than the above, Fldigi-3.13AJ works great and let me work
and log the 3D20CR Conway Reef Expedition on CW, RTTY & Phone
for a new country.  My thanks to the fldigi development team for a
FB program.
