> Version 3.11AI
>
>     TX Control Code fix
>         * Fix for Ctrl-C, Ctrl-V etc when entered in Tx widget
>           Bug associated with MS implementation
>           Fix should be compatible with all target OS.
>     NBEMS
>         Added NBEMS.files master directory creation
>         Added auto transmit of named file in NBEMS.files/wrap/auto
> directory
>         filename : wrap_auto_file
>     Hamlib Exception Handling
>         * Removed all C++ exception handling from the class 'rigclass'
>           Modified hamlib.cxx method calling to accept traditional
>           C style error returns.
>     RigCAT mods
>         * on WIN32 platform - reinitialize commport when driver fails
>           to be responsive.  Associated with some cheap USB dongle
>           drivers on XP / Vista.
>         * Modified serial i/o to accommodate transceivers that send
>           unsolicited data streams - typical of Icom 7... series when
>           transceiver CI-V port is set to "Transceive ON".
>         * Corrected computed read after write delay.
>
> You should see improved performance of the rigCAT interface.  Please
> report any significant changes .. good or bad!
>
> Hamlib would crash fldigi on MS XP and Vista (only two I can test).  The
> Hamlib modification corrects that bug.
>
> NBEMS mod anticipates a change to the way that files are passed between
> fldigi, flarq, wrap and flics.  When you start 3.13AI it will create a
> new directory / folder in your home directory named NBEMS.files.  It
> will also create a set of sub-directories in NBEMS.files.
>
> Fldigi would crash on MS XP and Vista if any control code other than
> Ctl-T or Ctl-R was entered at the keyboard when the Tx entry widget had
> focus.  Please test this on your systems and let me know if you still
> see the seg fault (crash in MS vernacular).
