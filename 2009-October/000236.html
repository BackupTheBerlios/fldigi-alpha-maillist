Dick Kriss wrote:

> I wonder why the Mixer controls are only on the Linux version and if
> they would be useful in other versions?

Because there is no equivalent in Windows or otherwise for the standard 
emulated OSS mixer control set that is used by Fldigi under Linux.

It is this emulated OSS mixer functionality that translates the myriad 
of exotic mixer controls found on modern sound cards and card types into 
the standard mixer elements that Fldigi can control using it's mixer applet.

The Linux (ALSA) driver maintainer aliases the appropriate specific 
sound card driver controls to the ones that are appropriate on the 
specific sound card driver's emulated mixer controls under OSS 
emulation. These emulated controls are slaved to the actual mixer 
elements in the specific sound card driver in use.

Every sound card looks alike to Fldigi via the OSS emulated mixer. These 
emulated mixer controls move the appropriate real ones, so ALSA, 
PortAudio, PulseAudio, all work.

In the worst case, the user aliases these manually.

There is no equivalent in Windows.
