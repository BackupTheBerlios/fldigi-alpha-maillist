The windows fldigi has been built against a modified hamlib that is 
statically linked.  Change were made to prevent the program from trying 
to use existing hamlib dll(s).  If you had to remove a program that used 
hamlib to avoid dll hell you might be able to restore that program now.

If you are testing NBEMS in the Puppy environment, you will need both pets.
