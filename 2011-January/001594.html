If you're using hamlib, and you're willing di fiddle with it a bit, you
can:

- set up rpc.rigd or rigctld as hamlib backend
- have fldigi use the corresponding rig model to talk with rpc.rigd or rigctld
- set up the exec macro to call a tiny shell script that does what you
  want by means of calls to rigctl, this one also configured to talk
  with rpc.rigd or rigctld.
