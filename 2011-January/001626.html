Start flrig from a terminal, you hopefully will see some errors.

In flrig open the event log, set it to debug, let it run for a while and 
save the log. Send it to Dave.

Are you sure there is only one instance of flrig running?

You can also open flrig with the config-dir switch and see if you are 
getting the same behavior.

I would also look at the rig setup in fldigi, make sure there are no 
tick marks anywhere nut the xmlrpc tab.
