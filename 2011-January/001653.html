Dave/Group,
 
I have to agree with Dick on this one. It would be nice to be able to fully
specify the Rig Mode and all available options that are supported both in
Hamlib or flrig. Short of that type implementation, the QSY function should
be able to duplicate the function of a memorized entry in the frequency
list. Maybe even call up one of those (but there are no name tags yet, so
that's not possible).
 
The QSY function will be different than the GOFREQ call which would simply
change the rig frequency.
One feature of the QSY MACRO that I would like to see would be a relative
change such as QSY:+000.500 which would change frequency by a user specified
increment. This would allow for a nice instant QSY while in QSO to move away
from a nearby station. Combine this with a TxID:on and one end could QSY and
signal the other end to follow suit. Obviously capabilities would be limited
by the RIG control functions available.
 
The QSY macro has the capability to be a very powerful tool.

On Jan 23, 2011, at 6:38 PM, Joseph A. Counsil wrote:


Dick & Dave,

I tried the QSY macro and it works perfectly for me (WinXP Pro/SP3, Yaesu
FT-100D, RigCAT).

For example, let's say I have a regular net at 3580.515, DominoEX8, 1500 Hz
on the waterfall, then I would use the following macro:

<QSY:3580.515:1500><MODEM:DOMEX8>

For complete control, I still need additional macros to set rig mode (USB,
LSB, RTTY, DIG, etc.) and filter width in case I use a filter (which I do).
This is a good step forward, and I can make some button sets for various hot
spots/modes I like to go to quickly.  With the additional rigmode and
filterwidth macros, I could have something like this for complete macro
control:

Dreaming of...

<QSY:3580.515:2200><RIGMODE:DIG><FILWID:500><MODEM:DOMEX8>

Great work - Thanks Dave & crew!
