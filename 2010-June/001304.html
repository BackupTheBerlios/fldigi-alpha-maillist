I am back using fldigi-3.21.0AF on the iMac using OS X 10.6.4.  The fire
drill last week was the result of a corrupt fldigi_def.xml file. Like the
old 
bumper sticker says...it happens.

I conducted some pretty extensive tests this morning going from a solid
working 3.20.17 to 3.21.0AF and then back to 3.20.17 with no issues. I
was able to write and save changes to the fldigi_def.xml file.

I have learned (the hard way) to keep a good copy of my fldigi_def.xml
for when things get screwed up - again.

Yes, I know it is dangerous to run released and alpha versions on the same
setup.  Mike¹s suggestion to operate alpha as another user on the platform
was good; however, I prefer to run the most current alpha under my own
identity. 

It was a learning experience trying to remember all of my personal
configuration settings when I had to start over as new user.  I know my
way around fldigi pretty good and I had a hard time remembering what
panel controlled certain preferences.  Fldigi has lots of neat user options.
