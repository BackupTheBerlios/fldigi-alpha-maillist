Ok, I see you a re using 3.22.0CM... But what OS?

Your Audio device is the onboard audio card? PCI Card? USB dongle?

Re-reading your original post to the list I see this detail:

============%<=============================
Expression 'parameters->channelCount <= maxChans' failed in
'src/hostapi/alsa/pa_linux_alsa.c', line: 1438
Expression 'ValidateParameters( inputParameters, hostApi,

This error is like the audio-in have LESS audio channels that expected; 
I have read some here that some cheap USB dongles have only mono audio-in...

This can be the cause, but this is just a shot, not a C programmer here...

BTW, try to backup your config "mv ~/.fldigi ~/.fldigi-bkp" and start it 
and in the wizard pick any of the none pulse options and try to run it 
with that... (port audio will be my next choice, as Ubuntu has no OSS 
support)

if you want to restore your olf fldigi settings run this

"mv ~/.fldigi-bkp ~/.fldigi"
