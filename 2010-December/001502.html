There's been some thought abotu an fldigi-lib but it would be a major task.
Outside of fldigi I did a PSK engine (ui-free, network-based) in Java 
called znudigi, mostly for academic purposes; it doesn't do spotting or 
multi-decoding.

With the current software, here's what I found:
Setting the waterfall speed to PAUSE causes the CPU usage to drop from 
14% to 12% on my system.
Changing from Best to Fast Sinc interpolator under Sound Settings causes 
it to drop to 4%.
I also tried moving the main RX squelch slider all the way up and didn't 
see any change.
At that point, re-enabling waterfall speed didn't increase CPU usage.

What X server are you using? There is an X server called xvnc that 
writes solely to memory and not to display, and you can then connect to 
them with VNC if you want to see.
