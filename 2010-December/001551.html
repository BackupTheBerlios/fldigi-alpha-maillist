Version 3.21.0AW
      Alpha release

RTTY visible timeout
  Added code to display active RTTY channel for time out period after 
signal extinguishes.

Browser start frequency
  Change to single control for both Waterfall and Browser low frequency.

Macro Bar Configure
  Added code to allow user to select 2nd macro position above or below 
the Rx/Tx panels



No other choices folks.  WYSIWYG  :-)

A new install defaults to a plain vanilla wrapper ... no visible 
browser, 1 macro bar.

I see an increase in cpu usage with 3.21 over 3.20; 8% versus 6%.  I'll 
have to test further on my slowest machine to see if this will create a 
problem for some users.
