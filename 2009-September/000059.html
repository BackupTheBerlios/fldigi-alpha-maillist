There is some documentation in the online help, but it does not describe
some recent work on the notifier such as the RSID support.  For the
latter:

Select "RSID reception" from the event menu.

Check "Ignore duplicates" and set a suitable time span.  This will
ignore successive RSID transmissions for the previously received mode
and frequency (+/- the RSID precision).  Not essential but recommended.

Set trigger limit to 0 (no limit) unless there are a lot of RSID
transmissions on the band.

Click the button next to "Show alert window" to insert the default text.
Or click on the button next to "Append to RX text" to get the text that
inserts a clickable marker in the received text every time the event is
triggered.  You can customize the text and of course you can have both
action types.

Select an appropriate window timeout, e.g. 30s, if using the alert
window.

Test the event to see what the alerts do.  If you are ignoring
duplicates, you will find that subsequent tests will be ignored
according to the Dup. time.  To get around this you can reselect the
event from the list at the bottom to redisplay the event -- this resets
the timeout of the temporary event that is used for testing.  Or you can
enable duplicate filtering after testing :-)

Finally click the Add button to register the event.


With an RSID reception event set up, you could also select
"Notifications only" in the RSID pane of the main config.  The
notification windows will show the detected modem and frequency and the
"Go to ..." button will switch to that modem/freq.

Note: naturally, if you select "Notifications only" but have no RSID
events registered or they are all disabled (or they have no actions),
then RSID reception will be effectively ignored even if enabled in the
main window.
