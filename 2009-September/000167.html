This is a good day. I have fldigi-3.13AH functioning with
RigCAT to the Kenwood TS-570. Some minor tweaking
may be in order.

As some of you may recall I have been able to use Hamlib
with my Kenwood TS-570S but have never been able to
get RigCAT to work.

Rather than sending an attachment Dave sent me a
message with revised text for the TS-570 definition file
appended to his email message.  I copied and pasted in
his modified version

<STATUS>
    Verified
    Version:    3.13
    Date:        17 September 2009
</STATUS>

into my "TS-570" file and it still did not work. I slept on
the problem for a few day and just figured out the problem.

When I went to 
<http://www.w1hkj.com/xmls/kenwood/TS-570.xml>
per the online help page, it opened up to a text version
of the definition.

------
Kenwood TS-570 Dave Freese     W1HKJ Tested by:     Bob Christenson, WU9Q
Verified Version:    3.11 Date:     15 April 2009 RigCAT - TS-570 50 2 5
9600 true false true LSB31 USB32 CW33 FM34 AM35 FSK36 CW-R37 FSK-R39 LSB FSK
CW-R MODE 4 MD BYTE 1 ; FREQ 14 FA DECIMAL 11 99999999999 00001500000 1 ;
SETFREQ 14 FA DECIMAL 11 99999999999 00001500000 1 ; FREQ GETFREQ 3 FA; FREQ
SETMODE 4 MD BYTE 1 ; GETMODE 3 MD; MODE PTTON 3 TX; PTTOFF 3 RX;
-----

When I saved it from the Apple Safari browser, it saved
the definition file as "TS-570" without the .xml suffix.  This
file "TS-570" has been saved in my .fldgi/rigs folder without
the suffix for months.  When I did an OS X "Get Info" on the
file, it showed it the filename as TS-570.xml with the
"Hide Extension" box checked.

I modified the TS-570 file to include the version 3.13
definition from Dave W1HKJ and the file still did not work.
The "fix" was to uncheck the OS X Hide Prefix box so the
filename now reads "TS-570.xml" and it works.

The filename problem could have been avoided if I had
used the right-click to download the file rather than
a left-click that opened the file as text.

The file now functions and I only have one issue to
resolve to keep RigCAT working.  I am getting the following
error using the version 3.11 version:
