>>>>> On Wed, 02 Sep 2009 22:43:54 -0400, David Kleber <davek at salsgiver.com> said:

> Stelios,
> The RSID detection in 3.13AD seems to be working well under Windows 
> Vista here.  I had it kick in a few times while monitoring 40m.

> One thing I think might be really handy would be to have a check box on 
> the front end GUI for TX RSID.  During our emcomm nets, we like to put 
> the RSID out on the first few calls and then when we go for general 
> checkins, but during nets, having RSID running at the beginning of every 
> transmission takes up valuable time.  It's pretty cumbersome while 
> running a net to have to go into the menu system to disable RSID on TX.

Dave has implemented these TXRSID and RXRSID toggles as check boxes
(actually light buttons) on the main window.  I can see how they would
be useful but am wary of the christmas tree effect.  Maybe we could make
them all blink on 25 Dec. :)

We have also been thinking that a <TXRSID> macro would provide finer
control.  This macro would expand to nothing and cause an immediate RSID
transmission for the current modem and frequency.  You could even have a
dedicated macro button (and therefore keyboard shortcut) for it.
Perhaps then the transmit-RSID toggle would not need to be on the main
window because you would not have to use it that often -- you could turn
it off and use <TXRSID> when required.  Anyway, <TXRSID> is on my TODO
list...
