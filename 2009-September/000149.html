w1hkj wrote:
> Version 3.13AG
>     1.  rigCAT mods
>         Restructured i/o to allow use with broadcast transceivers.
>         Icom 7xxx series default to a CI-V server that broadcasts
>         serial data which disrupted fldigi serial data stream extract.
>         New code passed tests on Vista with IC-746Pro operating in
>         CI-V Transceive (server) mode.
>     2.  Serial i/o Stopbits
>         Added stopbits configuration for hamlib and rigcat serial i/o
>     3.  Dom/Thor mod
>         Modified squelch evaluator for increased sensitivity
>     4.  QRZ lookup
>         Added post thread startup delay to allow OS to set up
>         conditionals.
>
> Notes:
>
>    1. rigCAT mods should provide better cat i/o with all makes of
>       transceiver.  You may need to adjust your timing.  This is a
>       major rewrite to the rigCAT execution code.
>    2. You need to know whether your transceiver uses 1 or 2 stop
>       bits.  Icom 7xxx series are usually 1 stop bit.  Kenwoods are
>       usually 2 stop bits.   I would like to know what performance you
>       see with a TS50 or TS450.  This mod was targeted at the 2 stop
>       bit transceivers.
>    3. Squelch for DominoEX and Thor modes should be faster acting and
>       also closer in level to PSK modes.
>    4. This should correct the 2x - 1st time QRZ problem on MS and some
>       Linux distros.
>
