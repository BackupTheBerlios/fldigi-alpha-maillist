The problem appears to be a timing issue between starting the QRZ 
request thread and issuing the first request.  The first request fails 
every time on MS because of the time it takes the OS to actually begin 
thread processing.  That appears to be an issue with your Linux 
distribution.  I solved (or bypassed the problem) by added a delay 
between starting the thread and the first request.  This works fine on 
MS, with never a miss on the first QRZ request.  It should also be a 
solution on Slackware.

It will be included in the next alpha release.
