I thought I'd better make sure this problem didn't get overlooked.  The first alpha version of fldigi I tried was 3.13AE.  In that version and 3.13AF there is a problem with rigCAT with the Icom IC-756 Pro III.  fldigi will error out and crash dead if I touch the tuning dial on the radio while rigCAT is enabled.  This is apparently from the rigCAT code not handling the frequency broadcast initiated by the radio whenever frequency is changed at the radio.  This is a feature of the Icom protocol that allows other devices like the IC-PW1 amplifier and other second party equipment to track and automatically adjust themselves to the new transceiver frequency.  These transmissions are produced automatically by the radio and can be recognized by the destination address in the header specified as 00H instead of E0H as used to respond to rigCat transmissions.  Version 3.12.4 did not crash with this situation but it did pop up warning dialogs infrequently when turning the tuning dial on the radio or changing modes.  Turning of the CI-V "transceive" mode in the Pro III allows 3.13AF to operate without crashing but I wouldn't want to have to manually turn it off and on every time I use fldigi.

flrig does not have this problem so I have been running it with 3.13AF with rigCAT turned off successfully.  Dave did adjust his code in flrig to ignore the Icom broadcasts.  If the code is organized to allow for it, these broadcasts could be used to update the rigCAT frequency indication as they arrive.
