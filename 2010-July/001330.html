> Has anyone besides me tested the multi-user logbook capability in 3.21.0AJ?
>
> Test by opening two or more instances of fldigi. Be sure to give each
> it's own unique set of start up params:
>
> --config-dir yadayada (full pathname)
> --arq-server-port (default is 7322)
> --xmlrpc-server-port (default is 7362)
>
> This test can be performed on Linux and Windows by modifying the launch
> icon "Target:" entry
>
> After getting the two or more instances running set them all to use the
> same logbook. I suggest that you make a copy of your regular log and
> then use the copy for the tests. Display the logbook on all the
> instances. Add, update and/or delete records on any of the logbooks and
> note whether the entries change accordingly in the others.
>
