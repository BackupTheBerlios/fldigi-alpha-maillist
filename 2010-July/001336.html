I am using a brand new Flex 1500 also in Windows with the latest FLDigi.

I installed the N8VB virtual comport driver which gave me com ports 4 and 5.

In the "CAT Control" tab (of the PowerSDR 2.05 settings dialog) I
checked the "Enable CAT" check box and put in following :
Port ; 5
Baud : 19200
Parity : NONE
Data : 8
Stop : 1

I did NOT enable PTT control - though COM1 is in the Port field and
DTR  is selected (that may not be necessary to do)

I set the ID as : TS2000

The ID may work on other things such as PowerSDR but it works as
TS2000 for me with no problems.

In FLDigi I used Hamlib, made sure I checked the box at the very top
of the tab, and set the Rig to Kenwood TS-2000 (BETA)
I set the COM Port to : 4
NOTE that the comport in PSDR is 5 and FLDigi is 4
I set the Baud rate and Stop bits to 19200 and 1 just as in PSDR
The only other thing selected on the tab page is DTR and I'm not sure
if it is necessary but since everything works fine I leave it alone.

There may be other ways to accomplish the same but I know this works
fine for me and my 1500
