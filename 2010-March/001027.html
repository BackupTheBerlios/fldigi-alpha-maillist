>>>>> On Mon, 15 Mar 2010 22:05:07 -0400, Tony <dxdx at optonline.net> said:

> All,
> I'm using a Flex 5K with virtual audio cable. I'm getting the Fldigi 
> "PortAudio error: Invalid number of channels" message at start up. Both 
> Fldigi and VAC are using the same sample rate (48K) and the converter is 
> set to Medium Sync Interpolator. Correction is 0 PPM.

> The Flex and VAC work fine with other digital programs so not sure 
> what's going on.

I seem to recall that this error is related to the number of output vs. input
channels.  Try unchecking "Enable right audio channel" in Configure->Sound
Card->Settings.  This setting takes effect immediately, but if sound
initialisation has failed it may be safer to restart fldigi anyway.

If this doesn't work, please run fldigi with `--debug-level 9' and send us a
copy of the event log.  Copy the log from Help->Event log or find the file
status_log.txt in the folder opened via File->Folders->Fldigi config".
