Guys,

 

There seems to be some issues in the OSX version:

Opening the default browser for lookup on QRZ.com sends the right info to
QRZ, but does not always maximize the browser (firefox).

There is an issue with the audio out (port audio) on OSX. When TXiD is
selected and after changing modes from PSK modem to Olivia, between the
"normal" Olivia modem output and the TXiD identifier being sent, there is a
loud clicking transient (single burst). This tends to set off the RX level
indicator to RED (and it locks it in the RED condition) when monitor TX is
selected.

AFC (PSK modem) will not lock into an RX signal after QSY is selected, but
the same signal in the PSK Browser will track AFC fine. Expanding the
waterfall to X2 and recentering the tuning and the AFC will then track
properly. I checked the CAT functions and all works well using the Hamlib
back end.

 

I did notice that there were a few upgrades to Firefox over the past week,
which may explain the browser issues.

 

I plan to retest these issues with 3.20.0b5 later today.

 

Thanks,
