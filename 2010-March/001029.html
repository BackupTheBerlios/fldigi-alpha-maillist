Guys,

I just tried the new version of flrig(1.0.0b6) with my FT1000MP.
I got the CAT to keep running but the frequency display is somehow linked to
the selection for RX bandwidth and does not display the correct frequency.
During "normal" polling, it appears that a new frequency is constantly being
commanded and the RX  is being muted at the polling frequency (~10 Hz).

On exit (close program) there is a burst of data that commands the rig to
1.100000000 MHz.

We are close, but it appears the data formats are just not correct.
For use on most MAC systems the Window for the serial (tty) ports name drop
down is too short and the full name of the ports cannot be seen. This is the
same for the flrig version on MAC also. In my case the typical port name is:
/dev/tty.usbserial-FTDWGU46  (27 Characters).

The backend for Hamilb has been working fine in the complied in version.
Environment:
OS-X 10.6.2 on Intel MAC
CAT via external USB to serial
Second serial port for the RIGblaster (only using RTS+)
