Compiles OK in Ubuntu 9.10 system at W5YM

Using it with the TS-940 radio, using the TS-440 .xml file since there
isn't one for the 940, in Linux fldigi gets frequency from the radio but
cannot set the frequency of the radio.  In Windows XP the radio frequency
doesn't go in either direction, and there is an error box 
W:rigCAT_getfreq: Retries failed.  The log is continuously logging
that message alternating with W: rigCAT_getmode: Retries failed.
fldigi can set the mode of the radio, but does not get it if it is
change on the radio.

In Linux the mode information does go in both directions (you can change
the mode from the program or the radio and the other gets it).  There
is a single message logged  W: rigCAT_getfreq: Retries failed
and fldigi is getting the frequency from the radio as the knob on
the radio is turned.  But fldigi can't control the frequency of
the radio.  Which means the QSY button isn't useful, although it
doesn't matter much anyway with the TS-940 since the only filter
bandwidth available in SSB mode is the SSB filter.
