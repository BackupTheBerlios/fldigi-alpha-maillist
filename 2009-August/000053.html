> I suspect you somehow got an SVN revision of PowerSDR that had that
> old problem in it. Be sure to verify that you are getting the latest
> version of the binary from the test branch. I am currently running
> revision 3339.
>
> OTOH, if you are relatively new to Flex and PowerSDR, you should make
> everything work with the release version, i.e. 1.18.2, first. After
> you do that and make sure everything is working you can switch to
> trying the alpha versions. That way you can be sure that any problems
> you encounter are a function of the alpha code and not your initial
> set-up. Nothing is more frustrating than trying to debug two or three
> things at once.
>
> Do remember to recalibrate the receiver image rejection whenever
> switching from the production to the test branch software and
> vice-versa. One of the new features being tested in the test branch is
> an on-the-fly continuous image-rejection calibration that uses strong
> in-band signals instead of the built-in test oscillator.
>
> Good luck and happy digiting!
>
