Algorithms for detecting voice activity (Voice Activity Detection) are 
used on GSM phones and VOIP in order to stop emission of packets when a 
user is not talking, therefore reducing packets traffic.

Information here:
http://www.ciscopress.com/articles/article.asp?p=606583&seqNum=7
http://en.wikipedia.org/wiki/Voice_activity_detection

It is so commonly used that there is even an implementation in the Intel 
IPP library : 
http://software.intel.com/sites/products/documentation/hpc/ipp/ipps/ipps_ch8/ch8_voice_activity_detector.html#ch8_voice_activity_detector

Wonder whether such an algorithm could be of use in fldigi (Although 
it's a bit out of scope because not digital):
- Automatically setting the PTT when voice is detected on the input.
- A scanner for AM/FM based on voice/music detection.

Any idea ? Thanks.
