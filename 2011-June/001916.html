Running fldigi with command line parameters is a PIA on OS X!  You need 
to open a terminal window and then execute the actual fldigi executable 
(not the pretty OS X icon) in order to pass a command line parameter.  I 
was not able to figure out how to do otherwise.  Come on Apple give us 
break!

Anyway, I will probably be creating a logbook configuration tab for the 
remote logbook.  That will make like easier for both OS X and Windows 
users, neither of whom are very comfortable with the command line.

I can report that I have tested the client/server setup with my 
equipment as follows:

Logbook server - On Ubuntu 10.10 system
Fldigi client - On Ubuntu 10.10 system (same computer as server)
Fldigi client - On OS X, mini-mac G64
Fldigi client - On Ubuntu 11.04 system
Fldigi client - On Puppy OS, 4.x, Compaq Notebook.

All systems on a single LAN.  All running concurrently and successfully 
making queries to the server and updating the database.

If someone tests with the server running on Windows or OS X please 
report your results.
