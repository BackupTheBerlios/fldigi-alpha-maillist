* On 2011 26 Jun 14:28 -0500, Pierfrancesco Caci wrote:
> 
> Hello all,
> would there be any problem with other brands of radios if the order of
> the commands sent when changing band/mode from the frequency list were
> changed so that the band change is done before the mode change? The K3
> recalls the last mode used on that band when changing it, so the mode
> change must be done after it, or it gets lost. 
> I think the point to be changed is  qso_selectFreq() in rigsupport.cxx

To follow-on, Elecraft states that programs should allow ~500 mSec upon
a band change before being sent another command.  It seems that Fldigi
is rather aggressive in its polling rate and this may factor into mode
not being changed properly, at least with Hamlib.  I would like to see a
polling rate setting in Fldigi's Hamlib configuration that does not
require changing Hamlib write delay and post write delay values.
