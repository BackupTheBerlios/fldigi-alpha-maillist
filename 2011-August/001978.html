Hi all.

Ok, testing locally the macro and I found that the first symbol get hurt 
in my local repeater (dtmfkit RX on other radio and i always lost the 
first symbol)

So, the delay in my local repeater is too long or the TX to data time in 
DTMF is to short.

My local repeater is a homebrew type, actually is a two TK-760G radios 
back to back with a bird tone generator using a NE555 for tx end tail...

I don't know if this happens with professionals repeater systems, but it 
can, please check this...

Then I need a delay between the TX start and the DTMF signal... 
searching in the macros, the IDLE can help me...

I set up a macro like this:

<TX>
<IDLE:1>
<DTMF:80122416002>
<RX>

And it does not work... test it yourself, the IDLE is inserted *after* 
the DTMF tone, not before as it should be...try setting the delay to 2 
or 3 secs for increased effect...

So Dave and others, it will be useful that the DTMF macro signals obey 
the IDLE macro time.

This is for signals that pass *through* the repeater, not to the 
repeater system it self...

73
