Hi Dave, et. al.,

I had a chance to play with fldigi on a camping trip we took using my 
Yaesu FT-857D and here are my notes:

1)  It would be nice to have the option in fldigi's setup to use hamlib 
defaults for a particular rig.  I found that rigctl worked just fine, 
but at 4800 baud I needed 2.5 stop bits which isn't available.  It 
worked at 9600 baud with two stop bits.  I would have liked to try at 
38400, but it wasn't an option.   In conclusion, the parameters in the 
hamlib setup window need to include "hamlib default" (which you can see 
if you read the source code on any rig in hamlib).

2)  I liked the work around that you could "hardwire" the sideband 
used.  This allowed me to work around a hamlib deficiency that it didn't 
get the sideband correct in the "Digital" mode which was necessary for 
use with the audio inputs on the back of the rig.

Thanks for a great piece of software.
