Some design ideas to discuss:
- Reuse the existing code for sending/receiving SSTV images. (Right-click 
"Send image", "View/MFSK image" with the added feature of short-cutting 
apt=automatic picture transmission preamble). No X1/X2/X4 button though. 
Maybe reuse the automatic slant correction?
- Add a new modem type and a new tab in configuration window for setting 
parameters: Carrier, deviation, filter (Wide, narrow), apt start delay, apt 
stop delay, phasing lines etc... based on 
http://hamfax.sourceforge.net/Screenshot.html (Please note that Hamfax is 
GPL code, not maintained for more than five years).
- Add a new opMode with the following sub-opmode: IOC576, IOC288, colorfax 
(Not sure for this one), lpm=120 or 60
- Name the feature 'Weatherfax' because this is the keyword used in 
wikipedia ( http://en.wikipedia.org/wiki/Weatherfax ).

Btw it seems that weatherfax still seems to be widely used ships: Probably 
because it is very cheap, very reliable and with a very high safety added 
value.
