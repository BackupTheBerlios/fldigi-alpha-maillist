I have tried to duplicate the NO CAPS on Linux, Vista, XP and Puppy.  
Simply cannot do it.  However I can make the keyboard non-responsive.

If you left click in the Rx widget (that's Linux-speak for control) the 
cursor and the keyboard focus is assigned to that area of the gui.  The 
Rx widget rejects all keyboard entries.  So you can type away and get 
nothing.  Pressing the keyboard ENTER key will restore keyboard focus to 
the Tx widget.  Left clicking in the Tx widget will also do the same (be 
sure that the insert cursor is at the end of the text).  The TAB key 
moves the cursor from the current TX position to the end of text and 
reverse.  This is outlined in the on-line help.

So I will risk saying that there is a high probability that you had just 
transferred data from the Rx widget to the logging widgets.  Perhaps 
your last mouse click in the Rx area was a left click.  That would cause 
the effect reported by several testers.  It would also explain the 
seemingly random nature of the observation.
