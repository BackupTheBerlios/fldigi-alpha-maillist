  Version 0.9.96

    Improve transceiver response
        Changed from on demand to polled frequency updates.
        Prior behavior required that transceiver be sent every step
        change in the flrig frequency display.  New behavior only
        sends state at time of poll iff the value changes.  The poll
        is a part of the serial thread loop.  Original code gave
        poor response on low baud rate serial ports.

    Exception handler
        * Added catch (...) statements to correct "death by win32"
          problem on MS when closing down without the presence of
          a running fldigi.
        * Removed unused code from xml_io.cxx

All files updated including the pet.

73, Dave, W1HKJ
for the development team
