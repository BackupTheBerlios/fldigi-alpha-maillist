There are open-source implementations of Weather fax (
http://en.wikipedia.org/wiki/Weatherfax ) modems (Hamfax
http://hamfax.sourceforge.net/ or acfax
http://linux.maruhn.com/sec/acfax.html ); there would probably be a
benefit to integrate them into fldigi:

- Hamfax is not updated for a long time, has no hamlib support for PTT
command (I developed a patch to add that).
- Fldigi can provide a unique parameter interface, its waterfall
window can help tuning.
- Hamfax and acfax do not run under Windows, although the core part,
GPL, is pure C language (
http://hamfax.cvs.sourceforge.net/viewvc/hamfax/hamfax/src/ or
http://packages.debian.org/fr/source/sid/acfax  )

What is needed, among other things might be:
- Add a new modem and tab in fldigi configuration.
- Create a new macro for receiving/sending fax images.
- Create a new opmode.

So, is there an interest for a adding weather fax modem into Fldigi ?
