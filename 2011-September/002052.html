WWV xmt mode
       * Added sound card time tick transmit

Here is a macro which demonstrates how to use the WWV mode to transmit 
time ticks.

===========snip==========
<MODEM:NULL>
<!QSY:1802.0:1000>
<!MODEM:CW><!WPM:18>QST de <MYCALL>
1 min time tick cal run follows
de <MYCALL>
<!MODEM:NULL><!IDLE:2>
<!MODEM:WWV><!IDLE:60>
<!MODEM:NULL><!IDLE:2>
<!MODEM:CW>QST de <MYCALL> end of cal run de <MYCALL> k
<TX><RX>
===========snip==========

Change the <!QSY line (or delete it).  You might want to substitute 
<!GOFREQ:1000>

Sends a CW preamble announcing a 1 minute time tick calibration run
Sends time ticks for 1 minute. The pulses are 200 msec in duration with 
4 msec leading and trailing edges.  The pulse modulates a 1000 Hz audio 
sub carrier.
Sends a CW postample announcing the end of the calibration run.

Usage:
Master station calibrates the sound card to WWV on 5, 10 or 15 MHz (or 
the Euro / Asian counterpart).

Master sends a long time tick transmission which is copied by all other 
stations that want to be synchronized to the Master station.  The 
receiving stations copy the transmission just as if it were from WWV.  
The calibration procedure is identical.  This can be used to assist 
VHF/UHF FM stations who do not have an HF receiver to use for the WWV 
calibration of the sound card clock.  The sound card clock's error in 
ppm is measured by comparing receiving the once per second tick signal 
and adjusting for no slant on the WWV scope display.

I've posted an example wav for you to listen to: 
http://www.w1hkj.com/alpha/fldigi/timetick.wav
