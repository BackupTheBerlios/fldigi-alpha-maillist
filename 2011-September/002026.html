cty.dat is not accessed by fllog, only fldigi.  So that file belongs in 
the active fldigi files folder, i.e.

~/.fldigi
c:\Documents and Settings\<user>\fldigi.files
etc.
> While I was looking, I noticed that the log text .adif file had nothing
> in it, although the gui panel had all the remotely enterd logs
> appearing.
The file text.adif is used only for interfacing to external logging 
programs that require a file interface.  The database is updated 
immediately when a new entry is made (either at fllog or fldigi), or it 
is modified at fllog.  If this is occurring with the main adif logbook 
file then you might be seeing a permissions problem ... see paragraphs 
below.
