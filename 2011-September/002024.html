You've named a directory the same as an executable Graham.  I'm not sure 
if that is a cause of the problem

executable ==> fldigi
directory ==> ~/fldigi

Suggest you change your directory structure to use something like

~fldigi.general

and then make sure that the cty.dat file is in that directory (not a 
sub).  Also be sure that the cty.dat format is the one suitable for fldigi.
