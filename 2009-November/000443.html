Hi List,

This just to let you know that in hamlib 1.2.9 and to 1.2.10 the Mode change 
with IC 706 MKIIG is broken.

I ignore about other rigs.

You can check it from fldigi trying to change mode: only AM  and FM do succeed 
but it inserts the narrow filter ( M3 FIL).

This can be checked also using the rigctl ( In my case: rigctl -m 
311 -r /dev/ttyS0 -s 9600 -c0x58 -vvvvv)

Actually hamlib sends :

FE FE 58 E0 06 01 02 FD

or
 
FE FE 58 E0 06 03 02 FD

where the 02 forces the narrow band filter....this happens which ever 
parameter is given to the M command e.g.

M USB 1
or
M USB 2

Only M USB 0  fails which, in accord to the manual,  should set the default 
mode.

By the way I am using 'jpnevulator --ascii  --tty /dev/ttyS0 --read' to check 
what hamlib sends to the radio.

The same behaviour happens using rigctld as in DBLog.

Please, can somebody with other radio, check how is the behaviour of hamlib in 
his set-up ?

By the time I suggest to go back to version 1.2.8.

Also the polling of the mode seems to fail in USB and CW: the inserted filter 
is not recognised with the 'm' command, but it works if the radio is in AM o 
FM.
