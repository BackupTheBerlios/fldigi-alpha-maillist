Andy,

This is the intended behaviour with the latest server and Fldigi 3.13. The
server should listen to RSID (it runs in the background in version 3.13) and
switch to the requested mode automatically. The server automatically adjusts
it's internal timings according to the new mode.

The concept is to let the client choose the most appropriate mode for the
conditions and connect in that mode.

At the conclusion of the session, the server returns to it's designated mode
if you have it in frequeny or channel scan or mode scan mode.

I have attached a word document that will be included in the wiki that
explains the concept and has a setup section for the client and the server.

Hope this helps,

Best 73s, John
