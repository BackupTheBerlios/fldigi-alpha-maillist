=================== fldigi =====================
Version 3.13BW

    ARQ abort
        Modified ARQ abort - ARQ text buffer did not clear properly
        at end of transmission for long postamble modem types such
        as MT63 and Olivia.

    Update manual pages

    Fix -iconic option handling
      * Hide all windows if the -i/-iconic option was passed
      * Use Fl_Window::show(int, char**) to show the icon window (X11)

    Fix main window title modification
      * Append text to main window title such that the title set with
        -title is preserved as a prefix.
      * Use the -title argument as the JACK or PulseAudio stream name.

    Improve audio initialisation
      * Add code to check if the pulseaudio daemon is running
      * Prefer PulseAudio if available and no previous setting exists
      * Don't complain if PortAudio did not find any devices
      * If PortAudio initialisation fails we change the backend to
        "File I/O" only if it was previously set to PortAudio.

    Fix PulseAudio backend
      * Flush capture stream after transmitting
      * Reduce stream latency

    Fix OSS compilation on NetBSD
      The SNDCTL_DSP_{SYNC,POST} ioctls calls must pass a pointer as the 
third
      argument to avoid a compilation error on NetBSD, which specifies argp
      as a char*.

    Fix compiler warnings

    Remove -fomit-frame-pointer flag
      This seems to give only a very small speedup (1-3% in 
DominoEX/Thor and
      Olivia) and makes debugging impossible on x86.

    Configure Dialog
        Change appearance of unselected items on rig control tabs

    Olivia Divide by Zero Error
        Divide by zero occurred if 2-tone Olivia mode selected.

=================== flrig ==================
Version 0.9.99L
   
    XMLRPC sync delay
        Added 200 msec delay between initial set modes / bandwidths and
        set mode / bandwidth.  Needed to prevent premature response to
        xmlrpc mode/bandwidth query.

    OS X missing include
        Added <glob.h>

    WIN32 build fix

    Status File Change
        Removed rig_nbr from status file
        Determine rig_nbr based on noun name of last transceiver used.
        Status file is backward compatible.

    Serial Port Discovery
        Changed serial port discovery routine.
          * if available, use non-polled device lookup in Linux
          * added code to support OS X, FreeBSD serial ports

    TT-570 update
        Added TT-570 to enum declaration in rigbase.h

    TT-538 update
        Update to volume control read/write

    OS X exit on start
         Fix to OS X version exiting immediately after start

    TS570 bandwidth control and basic metering
    TS-570 S-meter scale adjusted
    Add initial Kenwood TS-570 support
    Kenwood TS rigs and Elecraft K2/K3 are all similar and yet different

    Use modulo arithmetic on loopcount in serial loop
