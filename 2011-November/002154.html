> fldigi:
>
> Version 3.21.27 - Maintenance release
>
> Log r/w improvements
> * Created separate thread for writing database. User with over
> 35,000 records was experiencing UI interruption of several
> seconds on Windows XP system.
> * Removed compatibility code from log r/w. Logs prior to 3.1 will
> read but some fields will be ignored.
> * Added new method "newrec" to CQsoDb class.
> * Modified logbook read and write behavior to eliminate duplication
> and employ faster string lookup algorithm in matching ADIF tags
> to internal database entries.
>
> WEFAX error logging fix
> * Fixed format compiler warnings and errors.
>
> Restore Log Sort
> * After adding a record
>
> fllog:
>
> Version 1.1.0 - Upgrade release
>
> R/W speedup
> * Changes to increase speed of ADIF file read/write
> * Added annunciation of read and write progress
>
> Both have been tested locally on Linux, XP, Vista and OS X with a log
> database of 106,000+ records.
>
> If you have a very large logbook database then you should consider using
> the fllog logbook server and fldigi as the client. That provides the
> absolutely fastest lookups, adds, amends, deletes etc without
> interrupting fldigi operation.
