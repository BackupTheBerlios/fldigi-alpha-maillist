I reported or requested these on the devel list after operating in CW
Sweepstakes.

The log change could be addressed by export, but I noticed that the adif
log file was storing the qso by the date you started but the time you
ended.  So if you have a qso which crosses a gmt date boundary it appears
in the way wrong spot in the file.  I reported to Dave that it was sorting
by date on and time off.

In a contest you probably want date on instead of date off, but as you all
point our, you can handle the choice in export.  No matter what the fix,
though, date on and time off together is wrong.

The clear tx macro is for use in other macros, not for use by itself.   I
wanted the contest macros to clear any accidental TX text before
transmitting.  The default macros CQ etc probably don't want to do this
since people might type in som,e text and press CQ instead of CALL or
something, but in general, I think contesters will want the macros to send
exactly only the text in the macro.

There is another bug that exec macros are inserting spaces even if there
is no output, but that is I believe unaddresed.
