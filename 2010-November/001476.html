TNX to Dave, W1HKJ, for the new user time options added in alpha fldig-3.20.30 and 3.21.0AO. They solve some long standing time related issues. The new options are in the User Interface General configuration panel. 

The first option checked by default is:

  Sort by Date/Time OFF

This option toggles the time used in the listing (display) of QSO's in the Logbook. In the default OFF mode, all QSO's are listed and sorted by the logged DATE and TIME_OFF. If the user prefers to display calls using the logged TIME_ON, the option should be unchecked. The export options will follow the user selected preference.

The other new option that is uncheck by default is:

  Date time ON==OFF

When checked, the time ON for new calls logged will be changed to equal the logged time OFF. This option has value to users, like me, who prefer to log QSO's based on TIME_OFF.

By having TIME_OFF equal TIME_ON, the ARRL's LoTW tqsl.app will convert the exported ADIF TIME_ON (really time off)to QSO_TIME.  In addition, this solves an issue with my OS X logging program, Aether, that converts TIME_OFF to QSO_TIME. My fldigi imports to Aether will now agree with my uploads to LoTW.

I do not want to enter a debate of logging Time ON vs. Time OFF. Dave has given fldigi users the option to use either. 

I had another example today when ZL8X came up on 20 meter phone. I selected his call on 20-Nov at 1415z and that time was displayed in the fldigi On box. I called, had lunch, listened for him to finally get to 5's and made the QSO at 1505z. When he confirmed the QSO, I clicked the fldigi log macro and the QSO is logged On and Off at 1505z. I feel sure ZL8X will have the QSO logged at 1505z. If I logged the old way the TIME_ON would be logged at 1415z resulting in a busted or orphaned QSO when uploaded to LoTW.

Yes, I could have used the fldigi hover-mouse-push over time on option to force the ON time equal OFF time. I have a hard time remembering to use this feature for each QSO.

Thanks Dave, the two new user time options will be useful and used by some of us.
