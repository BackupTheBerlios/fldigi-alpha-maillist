The problem as I see it is that we have these similar-but-different fixed
modes; e.g. olivia, contestia, MFSKx, DominoEX, and then the various flavors
of PSK; that make multiple changes but don't allow us to evaluate the
protocols while only changing one parameter at a time. That is certainly
useful for hams wondering which mode to try now but isn't helpful in
selecting which mode is likely to perform best. Where *is* the crossover
between IFK, MFSK, and PSK?

Hmm, wouldn't it be nice if the transmission header contained a descriptor
of the MODEM/CODEC so that the receiver could put together the correct
pieces.
