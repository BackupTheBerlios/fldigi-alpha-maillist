I have a fully automated solution, but it's linux-only...

I don't use the fldigi logbook, instead fldigi automatically exports to xlog.

My own code then monitors the xlog file, and when it sees changes it uploads them to eqsl and lotw. (eqsl isn't that hard, but getting tqsllib running was more fun....)

The code is python, and so I suspect it would be simple to make it monitor the fldigi file - as long as fldigi writes each entry out as it is logged. I just find xlog suits my needs. 

Look at http://www.whabbit.demon.co.uk/xlog-uploader-0.2.tgz
